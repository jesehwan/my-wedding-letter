# 웹 배포

## Context
결혼식 안내 웹페이지(Next.js 16 + React Three Fiber)를 웹에 배포하려 한다.
3D 모델 에셋 ~65MB, 이미지 ~10MB. 현재 배포 설정 없음.

## 배포 옵션 비교

| 항목 | **Vercel** (추천) | Netlify | Cloudflare Pages |
|------|-------------------|---------|------------------|
| Next.js 지원 | 네이티브 (만든 회사) | 어댑터 필요 | 제한적 |
| 설정 난이도 | 최소 (거의 제로) | 보통 | 보통 |
| 무료 티어 | 100GB 대역폭/월 | 100GB 대역폭/월 | 무제한 대역폭 |
| 대용량 에셋 | CDN 자동 캐싱 | CDN 자동 캐싱 | CDN 자동 캐싱 |
| 빌드 시간 제한 | 45분 | 15분 | 20분 |

### 추천: Vercel
- Next.js를 만든 회사 → 호환성 이슈 없음
- `git push`만 하면 자동 빌드·배포
- 무료 티어로 결혼식 안내 페이지 트래픽 충분
- 추가 설정 파일 불필요

## 실행 단계 (Vercel)

### 1. 변경사항 커밋 & 푸시
- 현재 수정된 파일들 커밋
- GitHub 리모트에 푸시

### 2. Vercel 연동
```bash
# Vercel CLI 설치
npm i -g vercel

# 프로젝트 루트에서 실행
vercel
```
- Vercel 계정 로그인 (GitHub 계정 연동)
- 프로젝트 설정 자동 감지 (Next.js)
- 첫 배포 완료 → `*.vercel.app` URL 발급

### 3. (선택) 커스텀 도메인
- Vercel 대시보드에서 도메인 연결 가능
- 또는 무료 `*.vercel.app` 서브도메인 사용

## 주의사항
- `public/models/` (55MB) 가 Git에 포함되어야 배포됨 → 현재 일부 모델이 unstaged 상태
- `.env` 파일이 있다면 Vercel 대시보드에서 환경변수 별도 설정 필요
- 무료 티어 제한: Serverless Function 실행 시간 60초, 대역폭 100GB/월

## 검증
- `vercel` 명령 후 발급된 URL 접속
- 랜딩 페이지 → 3D 씬 진입 → 모델 로딩 정상 확인
- 모바일에서도 접속 테스트
